# Topic Template

Use this structure for every problem or topic note in the DSA repo.

1. Problem Statement
2. Constraints
3. Intuition
4. Pattern Recognition
5. Brute Force
6. Better Solution
7. Optimal Solution
8. Dry Run
9. C++ Implementation
10. Time Complexity
11. Space Complexity
12. Common Mistakes
13. Edge Cases
14. Interview Follow-ups
15. Similar Problems
16. Key Takeaways

## Problem note checklist

- Add the LeetCode problem number when available
- Mention the core pattern used
- Include a short revision sheet at the end
- Keep the explanation in C++
- Add 1 to 3 related practice problems per note
